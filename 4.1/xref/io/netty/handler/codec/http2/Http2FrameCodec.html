45  /**
46   * <p><em>This API is very immature.</em> The Http2Connection-based API is currently preferred over this API.
47   * This API is targeted to eventually replace or reduce the need for the {@link Http2ConnectionHandler} API.
48   *
49   * <p>An HTTP/2 handler that maps HTTP/2 frames to {@link Http2Frame} objects and vice versa. For every incoming HTTP/2
50   * frame, an {@link Http2Frame} object is created and propagated via {@link #channelRead}. Outbound {@link Http2Frame}
51   * objects received via {@link #write} are converted to the HTTP/2 wire format. HTTP/2 frames specific to a stream
52   * implement the {@link Http2StreamFrame} interface. The {@link Http2FrameCodec} is instantiated using the
53   * {@link Http2FrameCodecBuilder}. It's recommended for channel handlers to inherit from the
54   * {@link Http2ChannelDuplexHandler}, as it provides additional functionality like iterating over all active streams or
55   * creating outbound streams.
56   *
57   * <h3>Stream Lifecycle</h3>
58   *
59   * The frame codec delivers and writes frames for active streams. An active stream is closed when either side sends a
60   * {@code RST_STREAM} frame or both sides send a frame with the {@code END_STREAM} flag set. Each
61   * {@link Http2StreamFrame} has a {@link Http2FrameStream} object attached that uniquely identifies a particular stream.
62   *
63   * <p>{@link Http2StreamFrame}s read from the channel always a {@link Http2FrameStream} object set, while when writing a
64   * {@link Http2StreamFrame} the application code needs to set a {@link Http2FrameStream} object using
65   * {@link Http2StreamFrame#stream(Http2FrameStream)}.
66   *
67   * <h3>Flow control</h3>
68   *
69   * The frame codec automatically increments stream and connection flow control windows.
70   *
71   * <p>Incoming flow controlled frames need to be consumed by writing a {@link Http2WindowUpdateFrame} with the consumed
72   * number of bytes and the corresponding stream identifier set to the frame codec.
73   *
74   * <p>The local stream-level flow control window can be changed by writing a {@link Http2SettingsFrame} with the
75   * {@link Http2Settings#initialWindowSize()} set to the targeted value.
76   *
77   * <p>The connection-level flow control window can be changed by writing a {@link Http2WindowUpdateFrame} with the
78   * desired window size <em>increment</em> in bytes and the stream identifier set to {@code 0}. By default the initial
79   * connection-level flow control window is the same as initial stream-level flow control window.
80   *
81   * <h3>New inbound Streams</h3>
82   *
83   * The first frame of an HTTP/2 stream must be an {@link Http2HeadersFrame}, which will have an {@link Http2FrameStream}
84   * object attached.
85   *
86   * <h3>New outbound Streams</h3>
87   *
88   * A outbound HTTP/2 stream can be created by first instantiating a new {@link Http2FrameStream} object via
89   * {@link Http2ChannelDuplexHandler#newStream()}, and then writing a {@link Http2HeadersFrame} object with the stream
90   * attached.
91   *
92   * <pre>
93   *     final Http2Stream2 stream = handler.newStream();
94   *     ctx.write(headersFrame.stream(stream)).addListener(new ChannelFutureListener() {
95   *
96   *         @Override
97   *         public void operationComplete(ChannelFuture f) {
98   *             if (f.isSuccess()) {
99   *                 // Stream is active and stream.id() returns a valid stream identifier.
100  *                 System.out.println("New stream with id " + stream.id() + " created.");
101  *             } else {
102  *                 // Stream failed to become active. Handle error.
103  *                 if (f.cause() instanceof Http2NoMoreStreamIdsException) {
104  *
105  *                 } else if (f.cause() instanceof Http2GoAwayException) {
106  *
107  *                 } else {
108  *
109  *                 }
110  *             }
111  *         }
112  *     }
113  * </pre>
114  *
115  * <p>If a new stream cannot be created due to stream id exhaustion of the endpoint, the {@link ChannelPromise} of the
116  * HEADERS frame will fail with a {@link Http2NoMoreStreamIdsException}.
117  *
118  * <p>The HTTP/2 standard allows for an endpoint to limit the maximum number of concurrently active streams via the
119  * {@code SETTINGS_MAX_CONCURRENT_STREAMS} setting. When this limit is reached, no new streams can be created. However,
120  * the {@link Http2FrameCodec} can be build with
121  * {@link Http2FrameCodecBuilder#encoderEnforceMaxConcurrentStreams(boolean)} enabled, in which case a new stream and
122  * its associated frames will be buffered until either the limit is increased or an active stream is closed. It's,
123  * however, possible that a buffered stream will never become active. That is, the channel might
124  * get closed or a GO_AWAY frame might be received. In the first case, all writes of buffered streams will fail with a
125  * {@link Http2ChannelClosedException}. In the second case, all writes of buffered streams with an identifier less than
126  * the last stream identifier of the GO_AWAY frame will fail with a {@link Http2GoAwayException}.
127  *
128  * <h3>Error Handling</h3>
129  *
130  * Exceptions and errors are propagated via {@link ChannelInboundHandler#exceptionCaught}. Exceptions that apply to
131  * a specific HTTP/2 stream are wrapped in a {@link Http2FrameStreamException} and have the corresponding
132  * {@link Http2FrameStream} object attached.
133  *
134  * <h3>Reference Counting</h3>
135  *
136  * Some {@link Http2StreamFrame}s implement the {@link ReferenceCounted} interface, as they carry
137  * reference counted objects (e.g. {@link ByteBuf}s). The frame codec will call {@link ReferenceCounted#retain()} before
138  * propagating a reference counted object through the pipeline, and thus an application handler needs to release such
139  * an object after having consumed it. For more information on reference counting take a look at
140  * https://netty.io/wiki/reference-counted-objects.html
141  *
142  * <h3>HTTP Upgrade</h3>
143  *
144  * Server-side HTTP to HTTP/2 upgrade is supported in conjunction with {@link Http2ServerUpgradeCodec}; the necessary
145  * HTTP-to-HTTP/2 conversion is performed automatically.
146  */

227     private void tryExpandConnectionFlowControlWindow(Http2Connection connection) throws Http2Exception {
228         if (initialFlowControlWindowSize != null) {
229             // The window size in the settings explicitly excludes the connection window. So we manually manipulate the
230             // connection window to accommodate more concurrent data per connection.
231             Http2Stream connectionStream = connection.connectionStream();
232             Http2LocalFlowController localFlowController = connection.local().flowController();
233             final int delta = initialFlowControlWindowSize - localFlowController.initialWindowSize(connectionStream);
234             // Only increase the connection window, don't decrease it.
235             if (delta > 0) {
236                 // Double the delta just so a single stream can't exhaust the connection window.
237                 localFlowController.incrementWindowSize(connectionStream, Math.max(delta << 1, delta));
238                 flush(ctx);
239             }
240         }
241     }

247     /**
248      * Handles the cleartext HTTP upgrade event. If an upgrade occurred, sends a simple response via
249      * HTTP/2 on stream 1 (the stream specifically reserved for cleartext HTTP upgrade).
250      */
251     @Override
252     public final void userEventTriggered(final ChannelHandlerContext ctx, final Object evt) throws Exception {
253         if (evt == Http2ConnectionPrefaceAndSettingsFrameWrittenEvent.INSTANCE) {
254             // The user event implies that we are on the client.
255             tryExpandConnectionFlowControlWindow(connection());
256 
257             // We schedule this on the EventExecutor to allow to have any extra handlers added to the pipeline
258             // before we pass the event to the next handler. This is needed as the event may be called from within
259             // handlerAdded(...) which will be run before other handlers will be added to the pipeline.
260             ctx.executor().execute(new Runnable() {
261                 @Override
262                 public void run() {
263                     ctx.fireUserEventTriggered(evt);
264                 }
265             });
266         } else if (evt instanceof UpgradeEvent) {
267             UpgradeEvent upgrade = (UpgradeEvent) evt;
268             try {
269                 onUpgradeEvent(ctx, upgrade.retain());
270                 Http2Stream stream = connection().stream(HTTP_UPGRADE_STREAM_ID);
271                 if (stream.getProperty(streamKey) == null) {
272                     // TODO: improve handler/stream lifecycle so that stream isn't active before handler added.
273                     // The stream was already made active, but ctx may have been null so it wasn't initialized.
274                     // https://github.com/netty/netty/issues/4942
275                     onStreamActive0(stream);
276                 }
277                 upgrade.upgradeRequest().headers().setInt(
278                         HttpConversionUtil.ExtensionHeaderNames.STREAM_ID.text(), HTTP_UPGRADE_STREAM_ID);
279                 stream.setProperty(upgradeKey, true);
280                 InboundHttpToHttp2Adapter.handle(
281                         ctx, connection(), decoder().frameListener(), upgrade.upgradeRequest().retain());
282             } finally {
283                 upgrade.release();
284             }
285         } else {
286             ctx.fireUserEventTriggered(evt);
287         }
288     }

290     /**
291      * Processes all {@link Http2Frame}s. {@link Http2StreamFrame}s may only originate in child
292      * streams.
293      */
294     @Override
295     public void write(ChannelHandlerContext ctx, Object msg, ChannelPromise promise) {
296         if (msg instanceof Http2DataFrame) {
297             Http2DataFrame dataFrame = (Http2DataFrame) msg;
298             encoder().writeData(ctx, dataFrame.stream().id(), dataFrame.content(),
299                     dataFrame.padding(), dataFrame.isEndStream(), promise);
300         } else if (msg instanceof Http2HeadersFrame) {
301             writeHeadersFrame(ctx, (Http2HeadersFrame) msg, promise);
302         } else if (msg instanceof Http2WindowUpdateFrame) {
303             Http2WindowUpdateFrame frame = (Http2WindowUpdateFrame) msg;
304             Http2FrameStream frameStream = frame.stream();
305             // It is legit to send a WINDOW_UPDATE frame for the connection stream. The parent channel doesn't attempt
306             // to set the Http2FrameStream so we assume if it is null the WINDOW_UPDATE is for the connection stream.
307             try {
308                 if (frameStream == null) {
309                     increaseInitialConnectionWindow(frame.windowSizeIncrement());
310                 } else {
311                     consumeBytes(frameStream.id(), frame.windowSizeIncrement());
312                 }
313                 promise.setSuccess();
314             } catch (Throwable t) {
315                 promise.setFailure(t);
316             }
317         } else if (msg instanceof Http2ResetFrame) {
318             Http2ResetFrame rstFrame = (Http2ResetFrame) msg;
319             int id = rstFrame.stream().id();
320             // Only ever send a reset frame if stream may have existed before as otherwise we may send a RST on a
321             // stream in an invalid state and cause a connection error.
322             if (connection().streamMayHaveExisted(id)) {
323                 encoder().writeRstStream(ctx, rstFrame.stream().id(), rstFrame.errorCode(), promise);
324             } else {
325                 ReferenceCountUtil.release(rstFrame);
326                 promise.setFailure(Http2Exception.streamError(
327                         rstFrame.stream().id(), Http2Error.PROTOCOL_ERROR, "Stream never existed"));
328             }
329         } else if (msg instanceof Http2PingFrame) {
330             Http2PingFrame frame = (Http2PingFrame) msg;
331             encoder().writePing(ctx, frame.ack(), frame.content(), promise);
332         } else if (msg instanceof Http2SettingsFrame) {
333             encoder().writeSettings(ctx, ((Http2SettingsFrame) msg).settings(), promise);
334         } else if (msg instanceof Http2SettingsAckFrame) {
335             // In the event of manual SETTINGS ACK, it is assumed the encoder will apply the earliest received but not
336             // yet ACKed settings.
337             encoder().writeSettingsAck(ctx, promise);
338         } else if (msg instanceof Http2GoAwayFrame) {
339             writeGoAwayFrame(ctx, (Http2GoAwayFrame) msg, promise);
340         } else if (msg instanceof Http2UnknownFrame) {
341             Http2UnknownFrame unknownFrame = (Http2UnknownFrame) msg;
342             encoder().writeFrame(ctx, unknownFrame.frameType(), unknownFrame.stream().id(),
343                     unknownFrame.flags(), unknownFrame.content(), promise);
344         } else if (!(msg instanceof Http2Frame)) {
345             ctx.write(msg, promise);
346         } else {
347             ReferenceCountUtil.release(msg);
348             throw new UnsupportedMessageTypeException(msg);
349         }
350     }

352     private void increaseInitialConnectionWindow(int deltaBytes) throws Http2Exception {
353         // The LocalFlowController is responsible for detecting over/under flow.
354         connection().local().flowController().incrementWindowSize(connection().connectionStream(), deltaBytes);
355     }

357     final boolean consumeBytes(int streamId, int bytes) throws Http2Exception {
358         Http2Stream stream = connection().stream(streamId);
359         // Upgraded requests are ineligible for stream control. We add the null check
360         // in case the stream has been deregistered.
361         if (stream != null && streamId == Http2CodecUtil.HTTP_UPGRADE_STREAM_ID) {
362             Boolean upgraded = stream.getProperty(upgradeKey);
363             if (Boolean.TRUE.equals(upgraded)) {
364                 return false;
365             }
366         }
367 
368         return connection().local().flowController().consumeBytes(stream, bytes);
369     }

371     private void writeGoAwayFrame(ChannelHandlerContext ctx, Http2GoAwayFrame frame, ChannelPromise promise) {
372         if (frame.lastStreamId() > -1) {
373             frame.release();
374             throw new IllegalArgumentException("Last stream id must not be set on GOAWAY frame");
375         }
376 
377         int lastStreamCreated = connection().remote().lastStreamCreated();
378         long lastStreamId = lastStreamCreated + ((long) frame.extraStreamIds()) * 2;
379         // Check if the computation overflowed.
380         if (lastStreamId > Integer.MAX_VALUE) {
381             lastStreamId = Integer.MAX_VALUE;
382         }
383         goAway(ctx, (int) lastStreamId, frame.errorCode(), frame.content(), promise);
384     }

386     private void writeHeadersFrame(
387             final ChannelHandlerContext ctx, Http2HeadersFrame headersFrame, final ChannelPromise promise) {
388 
389         if (isStreamIdValid(headersFrame.stream().id())) {
390             encoder().writeHeaders(ctx, headersFrame.stream().id(), headersFrame.headers(), headersFrame.padding(),
391                     headersFrame.isEndStream(), promise);
392         } else {
393             final DefaultHttp2FrameStream stream = (DefaultHttp2FrameStream) headersFrame.stream();
394             final Http2Connection connection = connection();
395             final int streamId = connection.local().incrementAndGetNextStreamId();
396             if (streamId < 0) {
397                 promise.setFailure(new Http2NoMoreStreamIdsException());
398 
399                 // Simulate a GOAWAY being received due to stream exhaustion on this connection. We use the maximum
400                 // valid stream ID for the current peer.
401                 onHttp2Frame(ctx, new DefaultHttp2GoAwayFrame(connection.isServer() ? Integer.MAX_VALUE :
402                         Integer.MAX_VALUE - 1, NO_ERROR.code(),
403                         writeAscii(ctx.alloc(), "Stream IDs exhausted on local stream creation")));
404                 return;
405             }
406             stream.id = streamId;
407 
408             // Use a Map to store all pending streams as we may have multiple. This is needed as if we would store the
409             // stream in a field directly we may override the stored field before onStreamAdded(...) was called
410             // and so not correctly set the property for the buffered stream.
411             //
412             // See https://github.com/netty/netty/issues/8692
413             Object old = frameStreamToInitializeMap.put(streamId, stream);
414 
415             // We should not re-use ids.
416             assert old == null;
417 
418             encoder().writeHeaders(ctx, streamId, headersFrame.headers(), headersFrame.padding(),
419                     headersFrame.isEndStream(), promise);
420 
421             if (!promise.isDone()) {
422                 numBufferedStreams++;
423                 // Clean up the stream being initialized if writing the headers fails and also
424                 // decrement the number of buffered streams.
425                 promise.addListener(new ChannelFutureListener() {
426                     @Override
427                     public void operationComplete(ChannelFuture channelFuture) {
428                         numBufferedStreams--;
429 
430                         handleHeaderFuture(channelFuture, streamId);
431                     }
432                 });
433             } else {
434                 handleHeaderFuture(promise, streamId);
435             }
436         }
437     }
438 
439     private void handleHeaderFuture(ChannelFuture channelFuture, int streamId) {
440         if (!channelFuture.isSuccess()) {
441             frameStreamToInitializeMap.remove(streamId);
442         }
443     }

488     @Override
489     protected void onConnectionError(
490             ChannelHandlerContext ctx, boolean outbound, Throwable cause, Http2Exception http2Ex) {
491         if (!outbound) {
492             // allow the user to handle it first in the pipeline, and then automatically clean up.
493             // If this is not desired behavior the user can override this method.
494             //
495             // We only forward non outbound errors as outbound errors will already be reflected by failing the promise.
496             ctx.fireExceptionCaught(cause);
497         }
498         super.onConnectionError(ctx, outbound, cause, http2Ex);
499     }

501     /**
502      * Exceptions for unknown streams, that is streams that have no {@link Http2FrameStream} object attached
503      * are simply logged and replied to by sending a RST_STREAM frame.
504      */
505     @Override
506     protected final void onStreamError(ChannelHandlerContext ctx, boolean outbound, Throwable cause,
507                                  Http2Exception.StreamException streamException) {
508         int streamId = streamException.streamId();
509         Http2Stream connectionStream = connection().stream(streamId);
510         if (connectionStream == null) {
511             onHttp2UnknownStreamError(ctx, cause, streamException);
512             // Write a RST_STREAM
513             super.onStreamError(ctx, outbound, cause, streamException);
514             return;
515         }
516 
517         Http2FrameStream stream = connectionStream.getProperty(streamKey);
518         if (stream == null) {
519             LOG.warn("Stream exception thrown without stream object attached.", cause);
520             // Write a RST_STREAM
521             super.onStreamError(ctx, outbound, cause, streamException);
522             return;
523         }
524 
525         if (!outbound) {
526             // We only forward non outbound errors as outbound errors will already be reflected by failing the promise.
527             onHttp2FrameStreamException(ctx, new Http2FrameStreamException(stream, streamException.error(), cause));
528         }
529     }

531     private void onHttp2UnknownStreamError(@SuppressWarnings("unused") ChannelHandlerContext ctx, Throwable cause,
532                                    Http2Exception.StreamException streamException) {
533         // It is normal to hit a race condition where we still receive frames for a stream that this
534         // peer has deemed closed, such as if this peer sends a RST(CANCEL) to discard the request.
535         // Since this is likely to be normal we log at DEBUG level.
536         InternalLogLevel level =
537             streamException.error() == Http2Error.STREAM_CLOSED ? InternalLogLevel.DEBUG : InternalLogLevel.WARN;
538         LOG.log(level, "Stream exception thrown for unknown stream {}.", streamException.streamId(), cause);
539     }
